# GitMaintainer v0.1.1 Release Notes

GitMaintainer v0.1.1 updates the GitHub pre-release artifacts with dependency
and scoring fixes from the first post-v0.1.0 maintenance pass.

PyPI publication remains deferred until Trusted Publishing is configured.

## Highlights

- Adds dependency count summaries for root-level `Cargo.toml` manifests.
- Counts Cargo runtime, build, dev, optional, and target-specific dependency groups.
- Tunes scoring so active repositories without GitHub Releases are not marked risky solely because they have an old pull request backlog.
- Keeps stale pull request and missing-release reasons visible as review context.

## Verification

- Local test suite passed.
- Package build verification passed.
- Live CLI smoke testing covered `rust-lang/cargo` text and JSON output.
- GitHub CI passed for Python 3.10, 3.11, 3.12, and package build jobs.

## Known Limits

- GitHub API rate limits apply.
- Private maintenance activity is not visible.
- Bus-factor estimate remains intentionally rough.
- Package manifest detection is root-level. Unsupported manifests are still reported as metadata without dependency counts.
- Package artifacts are attached to the GitHub pre-release, but PyPI publication is not configured yet.
