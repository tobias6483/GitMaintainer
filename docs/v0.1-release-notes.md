# GitMaintainer v0.1.0 Release Notes

GitMaintainer v0.1 introduces the first CLI for estimating whether a GitHub repository appears actively maintained.

This release is intended to ship as a GitHub pre-release. PyPI publication is
deferred until Trusted Publishing is configured.

## Highlights

- Maintenance status: Active, Slow, Risky, or Abandoned
- JSON output for scripts
- README badge generation
- `--version` output for installed CLI verification
- Public GitHub activity signals for commits, releases, issues, pull requests, default branch, archived status, fork status, recent commit author concentration, package manifests, and basic dependency counts
- Bounded GitHub API pagination with API budget metadata in JSON output and a low-budget warning in text output
- Local and CI package build verification for source and wheel artifacts

## Known Limits

- GitHub API rate limits apply.
- Private maintenance activity is not visible.
- Bus-factor estimate is intentionally rough in the first release.
- Package manifest detection is root-level. Dependency count parsing currently covers `package.json` and `requirements.txt` only.
- Package artifacts are attached to the GitHub pre-release, but PyPI publication is not configured yet.
